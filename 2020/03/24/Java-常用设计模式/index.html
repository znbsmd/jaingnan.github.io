Java 常用设计模式

Posted on 2020-03-24 | In Java 基础 |

1.单例模式

  • 静态初始化，饿汉方式。

  • 双重检测，懒汉方式。保证了变量的可见性（visibility）。被volatile关键字修饰的变量，如果值发生了变更，其他线程立马可见，避免出现脏读的现象

  • 单例注册表方式，Spring Bean

知识盲点：new Instance() 弱类型，通过Class.forName()提前加载类才能使用，低效率，只能调用无参构造。 new: 强类型。相对高效。能调用任何public构造。

2.工厂模式

Spring Bean通过Class.forName().newInstance();创建所有类并存到hashMap。

3. 代理模式

利用java反射机制，动态的生成了一个代理类，直接调用代理方法即可。AOP 就是动态代理机制。

4.责任链模式

类似与请假审批流程。正常就顺序执行if else 判断即可。但是类庞大，所有方法都集中在一个类，违反了 “单一职责原则”，测试和维护难度大。如果增加额外需求，也违反了 “开闭原则”。（扩展时代码尽不改变源代码，无需重新测试、审查。）使用场景是多个对象处理同一个请求。过滤器一般都使用责任链模式。

5. 适配器模式

类似转接头，如slf4j 桥接到其他日志框架。
类适配器例子 对象适配器同理

6. 观察者模式

类似 Spring 事件驱动，由一个事件生产者通过事件广播器创造线程任务推送给多个监听者。

7. 构造者模式

构造者模式，适用于一个对象有很多复杂的属性，需要根据不同情况创建不同的具体对象。
