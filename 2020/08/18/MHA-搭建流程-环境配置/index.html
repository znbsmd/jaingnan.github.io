MHA 搭建流程&&环境配置

环境版本：

  • CentOS 7.8 （4台）一主 2 从 一 MNA
  • MySQL 5.7.28

一、环境安装 ：

1. MySQL安装

  • 下载
wegt https://cdn.mysql.com/archives/mysql-5.7/mysql-5.7.28-1.el7.x86_64.rpm-bundle.tar
  • 解压
tar xvf mysql-5.7.28-1.el7.x86_64.rpm-bundle.tar
  • 移除系统自带的 mariadb
yum -y remove mariadb-libs
  • 按顺序安装，因为service服务依赖了common、libs、client
rpm -ivh mysql-community-common-5.7.28-1.el7.x86_64.rpm
rpm -ivh mysql-community-libs-5.7.28-1.el7.x86_64.rpm
rpm -ivh mysql-community-libs-compat-5.7.28-1.el7.x86_64.rpm
rpm -ivh mysql-community-client-5.7.28-1.el7.x86_64.rpm
rpm -ivh mysql-community-server-5.7.28-1.el7.x86_64.rpm
rpm -ivh mysql-community-devel-5.7.28-1.el7.x86_64.rpm
  • 初始化用户
mysqld --initialize --user=mysql
  • 查看初始密码
cat /var/log/mysqld.log | grep password
  • 启动服务
systemctl start mysqld
  • 开机启动
systemctl enable mysqld
  • 修改密码
SET PASSWORD = PASSWORD('123456');

2. 关闭防火墙 (防止外一都关下)

systemctl stop firewalld
systemctl stop iptables
systemctl disable firewalld.service

二、搭建主从

1. Master 配置

  • 修改my.cnf配置
[mysqld]
log_bin = mysql-bin # 开启binlog 并命名
server-id=1 # 服务唯一id
sync-binlog=1 # 即时同步
binlog-ignore-db=performance_schema # 忽略不同步的db
binlog-ignore-db=information_schema
binlog-ignore-db=sys
  • 重启服务
systemctl restart mysqld
  • 授权给从库
# 授权给从库复制功能
grant replication slave on *.* to root@'%' identified by '123456';
# 加所有权限给 root账号
grant all privileges on *.* to root@'%' identified by '123456';
# 刷新配置
flush privileges;
# 查看配置
show master status;

2. Slave 配置

  • 修改my.cnf配置
[mysqld]
server-id=2
sync-binlog=1 # 每次写入都同步到binlog
binlog-ignore-db=performance_schema # 忽略不同步
binlog-ignore-db=information_schema
binlog-ignore-db=sys
relay_log=mysql-relay-bin  #开启中继日志并设置名字
relay_log_purge=0
read_only=1
  • 重启服务
systemctl restart mysqld
  • 配置 主库信息，开始同步
change master to master_host='10.211.55.14',master_port=3306,master_user='root',master_password='123456',master_log_file='mysql-bin.000010',master_log_pos=1864;
  • 开启 slave
start slave; // 开启同步

3. 配置半同步复制

Master配置

  • 安装插件
install plugin rpl_semi_sync_master soname 'semisync_master.so';
show variables like '%semi%'
  • 设置全局变量
rpl_semi_sync_master_enabled=ON
rpl_semi_sync_master_timeout=1000
  • 重启服务
systemctl restart mysqld

Slave配置

install plugin rpl_semi_sync_slave soname 'semisync_slave.so';
  • 设置全局变量
# 开启
rpl_semi_sync_slave_enabled=ON
  • 重启服务
systemctl restart mysqld

检查半同步状态

  • 查看参数
show variables like '%semi%'
  • Master主库 MySQL日志确认(查看有 semi-sync 关键字 则启动同步成功)
tail -f /var/log/mysqld.log
>>>>
Start semi-sync binlog_dump to slave (server_id: 3), pos(mysql-bin.000003, 1187)

三 、配置MHA

1. ssh互通

  • 每台机器生成 rsa.pub （一直回车生成）

    ssh-keygen -t rsa

  • 生成到 authorized_keys

cat id_rsa.pub > authorized_keys
  • 通过scp 复制到各个节点并把自己的ras.pub追加到authorized_keys,保证4台机器上authorized_keys文件包含4个rsa 公钥，使得访问互通
scp 10.211.55.14:/root/.ssh/authorized_keys .
cat id_rsa.pub >> authorized_keys

2. 下载 MHA Manager and Node rpm 文件

  • github 搜索 mha4mysql 下载 Manager and Node rpm

  • MHA 节点安装 Manager and Node

    • 提前安装Manager and Node 所需要的依赖
    yum install perl-DBD-MySQL
yum install perl-Config-Tiny
yum install perl-Parallel-ForkManager
yum install perl-Log-Dispatch
  • 后两个依赖没有 可以添加eperl源
    wget -O /etc/yum.repos.d/epel.repo http://mirrors.aliyun.com/repo/epel-7.repo
  • 安装MHA
    rpm -ivh mha4mysql-node-0.58-0.el7.centos.noarch.rpm
rpm -ivh mha4mysql-manager-0.58-0.el7.centos.noarch.rpm
  • 主从 Node 节点安装
yum install perl-DBD-MySQL
rpm -ivh mha4mysql-node-0.58-0.el7.centos.noarch.rpm
  • 创建配置文件

MHA Manager 主节点 配置

mkdir -p /usr/local/masterha/app1
mkdir -p /var/log/masterha/app1

另外三个节点 配置

mkdir -p /usr/local/masterha/app1
  • Manager 配置 /etc/app1.cnf （使用时候去除注释）
[server default]
# 主库的mysql 配置
user=root
password=123456
# ssh 用户名
ssh_user=root
# 从库的mysql 配置
repl_user=root 
repl_password=123456
# ping间隔，用来检测master是否正常，默认是3秒，尝试三次没有回应的时候自动进行failover
ping_interval=1

manager_workdir=/usr/local/masterha/app1
manager_log=/var/log/masterha/app1/app1.log
remote_workdir=/usr/local/masterha/app1
master_binlog_dir=/var/lib/mysql

[server1]
hostname=10.211.55.14
# 为1 可成为主节点
candidate_master=1

[server2]
hostname=10.211.55.12
candidate_master=1

[server3]
hostname=10.211.55.11
# 不会成为主节点
no_master=1
  • Manager 检测SSH 
masterha_check_ssh -conf=/etc/app1.cnf
  • 检测 复制 功能
masterha_check_repl -conf=/etc/app1.cnf

  • 启动 

    nohup masterha_manager -conf=/etc/app1.cnf &> /var/log/masterha/manager.log &

  • 检测是否启动成功

masterha_check_status -conf=/etc/app1.cnf
  • 出现 app1 (pid:32734) is running(0:PING_OK), master:10.211.55.14,等相关关键字 证明启动成功

四、注意事项 & 踩坑

  • 修改配置文件切记重启mysql

  • 修改 slave change mangager 注意 stop slave;

  • ssh 配置完成后 ，最好 ssh root@10.211.55.14 互相检测下

  • 配置 MHA 要把 slave 节点 开启binlog配置

    log_bin=mysql-bin

  • 切换Master主节点后，原Master节点会成废弃节点，需要重新手动配置change manager 变成slave 节点

  • MHA切换主从后，如果再想切换 模拟测试 ，需要检查/usr/local/masterha/app1/app1.failover.complete文件是否存在，存在则删除。

  • MHA 切换失败的各种原因可以在 配置文件配置的manager_log中查看 如： /var/log/masterha/app1/app1.log
